Seam Carving

——— Overview ———

In this project, I implement the algorithm presented by Shai Avidan and Ariel Shamir in Seam Carving for Content-Aware Image Resizing (SIGGRAPH). Whereas typical cropping methods fail to consider content when resizing an image, this algorithm first removes the seams (connected paths of pixels running either vertically or horizontally across the image) which it deems of least importance. Below is a GIF of my code in action (ironically, although the seam selection process is still illustrated, ezgif.com automatically resized each frame to fit the initial image dimensions).

In implementing Seam Carving, I first determine the “importance” of each pixel, employing the image’s gradient magnitude as my energy function. Because this function uses Sobel filters to approximate the image’s partial derivatives, low cost pixels represent areas of little change. As such, this energy function prioritizes the removal of seams in homogeneous sections of the image. Below is an example of an image’s gradient magnitude.

Once every pixel was assigned a cost, I used backtracking to create a cumulative energy matrix before determining the continuous path (seam) which requires the least energy to traverse. This seam is then used to create a Boolean mask which, when applied to the original image, allows for the pixels’ removal. By repeating this process on the resized result, one can iteratively reduce an image’s width or height by a single pixel until the desired dimension is achieved. I ran the algorithm on various movie stills (sourced from film-grab.com). The results are featured below.

Because this process can be rather slow, I further divided the function into a precompute section and an online section. To implement the former, I resized the image until it was a single pixel in either width or height, saving each seams’ mask into a matrix before removal. With this matrix, the online method can reduce the image to any new dimension (in a matter of seconds) by simply applying the necessary number of masks in order.

——— Short Film ———

To produce this short, I wrote a program which would generate (and subsequently decompose) a block-based city from hand-modeled Faces (planes). The inputs are pictured below.

After readying colored versions of these Faces, the script populates each Street with randomly sized Buildings, themselves assemblages of Blocks. To create each Block, the six necessary Faces are determined, oriented, and united, in accordance with the Block’s relative position within the Building. A subset of these are initialized as rigid bodies with varying “active” frames.

A second (much shorter) script was used to prepare cherry blossoms on the hand-modeled tree for an nCloth simulation.
